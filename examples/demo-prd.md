# 用户管理需求

## 搜索

- P0 用户必须可以输入用户名并点击“搜索”，页面展示匹配用户，并调用 `GET /api/users`。
- 搜索结果需要包含用户名、角色和状态；没有匹配结果时显示“暂无数据”。
- 搜索后页码必须回到第 1 页。

## 重置、分页与排序

- P1 用户点击“重置”后必须清空关键词并重新加载第一页数据。
- P1 用户可以上一页、下一页浏览用户；页码不得小于 1。
- P2 用户可以按用户名升序、降序排列。

## 详情和权限

- P1 管理员可以查看用户详情，并调用 `GET /api/users/{id}`。
- P0 普通用户页面显示“普通用户无删除权限”且不展示删除按钮；连续两次直接调用 `DELETE /api/users/{id}` 均返回 403，用户数据保持存在。

## 导出和上传

- P1 用户可以导出 CSV 用户列表，下载文件不能为空，列名和数据必须与页面筛选条件一致。
- P1 用户可以上传 txt、png 或 jpg 文件，文件最大 1MB；成功后页面显示“上传成功”。

## 代码验收

- P0 与用户管理相关的 typecheck、lint 和 test 必须通过。
